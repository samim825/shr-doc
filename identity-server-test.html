Create New User
User Info

To create a new user you should create a json file of the user. User should contain following fields:

  1. id -> String Type and should be unique
  2. name -> String type
  3. email-> String type
  4. groups-> List of Roles
  5. profiles-> List of UserProfile
UserProfile should Contain
  • name -> This identifies the user profile type. This can be facility/mci-supervisor/provider/patient.
  • id -> ID of the facility/mci-supervisor/provider/patient. Patient ID is the health ID.
  • catchments-> List of String which contain locations id/code. Ids are 30, 3026, 302607
UserProfiles name fields can be
  • id for facility will be the hospital/clinic ID.
  • mci-supervisor -> Those users who have mci manipulate type roles. mci-supervisor id is the User Info ID.
  • provider- ( Can be a Doctor or who provides information ).
  • patient -> is a registered patient having a health id.

To create a new user we have to add a new user in Identity-Server

Firstly we have to write user details against user email in userDetail.properties file like this:

test-new-user@test.com={"id": "18001","name": "Local Facility Admin","email": "test-new-user@test.com","groups": ["Facility Admin","SHR User","MCI User"],"profiles": [{"name": "facility","id": "10019871","catchment": ["3026"]}]}

The Json format is : 

    
        {
            "Id":"18001",        // unique id
            "name":"TEST NEW USER",
            "email":"test-new-user@test.com",
            "groups":[
            "Facility Admin",
            "SHR User",
            "MCI User"
            ],
            "profiles":[
            {
                "name":"facility",
                "id":"10019871",
                "catchment":[
                    "3026" // locations_code from from locations.properties file
                ]
            }
            ]
        }

Secondly set password against user email in userPassword.properties file like this:

test-new-user@test.com=password

Finally set an authentication token against client_id of the user in the clients.properties file like this:

18001 = test-new-user_auth_token

where 18001 is the User ID that is defined in the userDetails.properties file and test-new-user_auth_token is the token. Token could be any text.

Sign In
UserCredentials to sign in
  1. email
  2. password
  3. clientId ( User id )
  4. Auth Token
POST: http://localhost:8084/signin

Headers

client_id : 18700,

X-Auth-Token : local-shr-system-admin_auth_token

client_id is the user id that has to be defined in the clients.properties. X-Auth-Token is the token that is provided against the client_id in the clients.properties

Body : 

    
        {
            "email": "local-shr-system-admin@test.com",
            "password": "thoughtwork"
        }

email: User email that is defined in the userDetails.properties password: User password that is defined in the userPassword.properties file against the user email.

Response : 

    
        {
            "access_token": "bdf2e1f9-ef49-4733-89cb-7ab3340c155f"
        }
User Creation challenges and possible solutions
  • We can’t create or update a user using api.
  • To create a new user we have to update the properties file. We have to add a json file and have to add some text in the properties file. Then we have to build the project after that it will be runnable. But after deploying our application, it will create hassle for us.
  • After deploying the application we can store the new user’s data in the map. But if we re-deploy the application all data will be destroyed, that's why it is not a right choice.
  • To resolve this problem we should use a relational database.
Location

Location is used to create a new user. Location id use as catchment in user profile

Locations Type:
  • division
  • district
  • upazila
  • paurasava
  • union
  • ward
How to create new location
A location json file contain
  • code-> String Type ->location type code from Hierarchy
  • id -> String type -> code and id is same
  • name-> String type
  • name_bn-> Name of location in bangla
  • type-> Location type
  • active-> boolean
  • updatedAt-> updated time
  • hierarchy -> Hierarchy of locations like this division-> district-> upazila-> paurasava-> union

Challenges and Possible Solutions of location
  • We can only get a particular location with id or can get a list of locations using api. But I can't update or create locations with the help of api.
  • To create a new location we have to create a json file under a folder. Following this process if we want to add new locations we have to build and deploy the applications again and again.
  • To resolve this problem we can use a database.

Facilities

Facilities are used to create a new user. In user profile If the profile's name is “facility” at that time a particular facility id is used on that profile.

How to create new facility
Facility json file contain:
  • name-> Name of the facility (String Type)
  • url-> URL link to the unique ID API resource for the facility
  • id-> The id must be universally unique (String Type)
  • active : true/false indicates whether the facility is active or not.
  • createdAt : when the facility was created.
  • updatedAt : when the facility was last updated.
  • coordinates : [lang, lat ]
  • identifiers
  • properties
    • List of services
    • List of locations
    • List of contacts
    • catchment -> Location id/code
Each external identifier consists of the following components:
  • agency: agency who created the code. ex) ministry of health, UNICEF, etc.
  • context: context/external system in which the agency is using the ID. eg) HMIS, DHIS2, HR
  • id: unique identifier
Each Location consists of the following components:
  • division_code
  • district_code
  • upazila_code
  • paurasava_code
  • union_code
  • ward_code
Each contact consists of the following components:
  • name
  • email
  • phone
  • mobile
  • fax
